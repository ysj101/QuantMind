# 日本株AI売買支援システム 仕様書 v0.1

> ヒアリング結果を構造化した初版。実装着手前に読み直して齟齬を修正するための草案として扱う。

---

## 1. 目的とコンセプト

### 1.1 基本コンセプト
個人投資家（ユーザー自身）が、**小型株のスイングトレード**で優位性を出すための意思決定支援システム。AIは売買シグナルを生成・提示するのみで、発注は人間が手動で行う「**提案型AIアナリスト**」。

### 1.2 既存のAI売買システムとの差別化
一般的な「価格データを学習して買いシグナルを出す」系とは一線を画し、以下2軸を中核差別化要素とする。

1. **反証可能性の強制（Falsifiability First）**
   LLMに「この判断が崩れるシナリオ」を必ず明文化させ、毎日全シナリオを再評価する。単なる買い提案ではなく「どうなれば撤退か」までが1パッケージ。
2. **小型株特有の非構造化情報の深読み**
   アナリストの目が届かない小型株だからこそ、PDF決算説明資料・役員情報・IR発信パターンといったテキスト重視の情報でLLMが価値を出せる。

### 1.3 主要ユーザー
- 個人投資家（ユーザー1名、自分専用）
- 運用資金規模: **20万円**
- 取引単位: **単元株（100株単位）のみ**

## 2. トレード戦略の前提

| 項目 | 設定 |
|------|------|
| 対象銘柄 | 時価総額500億円以下（小型株、やや広め） |
| 売買スタイル | スイング（数日〜数週間保有） |
| 最大同時保有銘柄数 | 3〜5銘柄（集中型） |
| シグナル生成頻度 | 引け後に1日1回 |
| 発注方式 | 提案のみ（手動で自分が発注） |
| 「トレードしない」判断 | 明示的に扱う。「本日は新規推奨なし」が正常な出力の1つ |

## 3. データアーキテクチャ

### 3.1 データソース優先順位

| カテゴリ | ソース | 備考 |
|----------|--------|------|
| 株価ヒストリカル（日足/分足） | まず yfinance / Stooq（無料）、将来 J-Quants に切替 | データ抽象層を挟み差し替え容易にする |
| 財務データ | **EDINET**（主軸） | 決算短信・有価証券報告書 |
| 適時開示・ニュース | **TDnet**（主軸） | 当日開示の一次情報 |
| 非構造化（差別化ソース） | 下記 3.2 参照 | システムの中核 |

### 3.2 非構造化情報ソース（差別化の核）

| ソース | 取得方法 | 抽出対象 |
|--------|----------|----------|
| 決算説明資料（PDF） | 各社IRページから自動収集（スクレイピング／機械的パス） | 事業進捗の定性ニュアンス、経営計画の強度 |
| 役員情報（EDINET有価証券報告書） | XBRL／テキスト抽出 | 社長経歴、株主構成、役員報酬構造 |
| IR発信メタデータ | TDnet／IRページの発信日時ログ | 発信**頻度・日程パターン**の変化検知（汚黙会社／急に活発化した会社） |

### 3.3 データストレージ
- **DuckDB + Parquet** を採用
- 分析クエリとバックテストの高速化、Parquet経由での外部連携容易性が理由
- ローカルファイルベース。サーバ不要

## 4. シグナル生成パイプライン

### 4.1 日次パイプライン（引け後実行）

```
[1] 市場レジーム判定
    ├─ VIX / 日経平均25日線 / 円ドル でRisk On/Off判定
    └─ Off日はシグナル生成を抑制

[2] ユニバース構築
    └─ 時価総額500億円以下 × 単元株で購入可能な価格帯で絞り込み

[3] ルールベース事前スクリーニング（LLMなし）
    └─ 出来高急増・IR発信あり・テクニカル異常などでTop N銘柄を抽出

[4] 非構造化情報の取得／更新
    ├─ 当日TDnet開示
    ├─ 直近決算説明資料PDF
    └─ 役員情報／IR発信頻度メタ

[5] LLMディベート（Bull vs Bear）
    ├─ Bull役（Claude Code または Codex）: 強気論を展開
    ├─ Bear役（もう一方）: 反論・リスクを指摘
    └─ 最終ジャッジ: 統合し推奨度を算出

[6] 反証シナリオの生成と更新
    ├─ 新規推奨にはシナリオを必ず添付
    └─ 保有銘柄・過去シナリオも全件再評価

[7] 保有銘柄のアクション判定
    └─ ホールド / 利食 / 損切 を提示

[8] PostMortem（クローズ銘柄がある場合）
    └─ 「予想 vs 結果」をLLMに書かせメモリ化（学習ループ）

[9] レポート生成
    ├─ PDF / HTML （ローカル保存、自分で開く）
    └─ CLI 出力
```

### 4.2 LLMディベート方式
- **Bull/Bearディベート方式** を採用
- **ローカル Claude Code と Codex** を対話させる（API費用ゼロ、ユーザーの既存ツールを活用）
- 片方がBull、もう一方がBearに回り、反論しながら統合ジャッジを出す
- LLMの役割はフル定性判断（テクニカル＋ファンダ＋開示文章を総合）

### 4.3 LLMコスト・スコープ方針
- **バックテストはルールベース部分のみで成立**させる。LLMディベートはライブ運用に限定
- ライブで評価した判断は DuckDB にログとして積み上げ、将来の特徴量化・学習再利用に備える

## 5. 差別化要素の詳細設計

### 5.1 反証可能性の強制

**目的**: 「買い提案だけ出すAI」に陥らない。崩れる条件を事前に言語化し、日次で状態を観測する。

**実装方針**:
- 新規エントリー推奨ごとに、LLMに以下を必ず生成させる:
  - **反証シナリオ**: 「このエントリーが誤っていたと判明する具体的事象」
  - **定量トリガー**: 可能な限り数値条件に分解（例: 「出来高が20日平均の50%を5日連続で下回る」）
  - **定性トリガー**: 数値化困難な事象（例: 「競合が類似の新製品を発表した」）
- 毎日、**全保有銘柄・過去の反証シナリオを再評価**
  - 定量トリガーは自動判定
  - 定性トリガーはLLMに当日ニュース・開示を渡して再評価（定性・定量ハイブリッド）
- トリガーヒット時はレポートで即「撤退推奨」としてエスカレーション

### 5.2 小型株向け非構造化情報深読み

**狙い**: アナリストカバレッジが薄い小型株で、「人間でも読むと情報価値があるが読むのが面倒な資料」をLLMに読ませる。

**具体実装**:
- **決算説明資料PDF**: 四半期ごとに各社IRからDL、LLMで「前期との定性変化」「経営陣の温度感」を要約化
- **役員情報**: EDINETから社長経歴・株主構成・役員報酬をテキスト抽出し、異常値（社長交代直後・大株主変動など）を検知
- **IR発信メタ観察**: 発信間隔・曜日パターン・時間帯の変化を時系列で追跡。「汚黙→活発化」などの兆候を自動検出

## 6. リスク・資金管理

| 項目 | ルール |
|------|--------|
| 最大同時保有銘柄数 | 3〜5銘柄 |
| 1銘柄あたり上限 | 資金20万円 ÷ 同時保有数 で算出（〜6.7万円／銘柄目安） |
| 取引単位 | 単元株のみ。1株単位（S株）は対象外 |
| マクロフィルタ | Risk Off日は新規シグナル生成を抑制 |
| 反証トリガー | ヒットしたら保有継続の再評価を強制 |

## 7. バックテスト

### 7.1 方針
- **ルールベース戦略部分**のみをバックテスト対象とする（LLM部分は除外）
- 過去データに対してLLMディベートは再実行しない（コストとスケーラビリティ）

### 7.2 評価指標
- **シャープレシオ**（最重視）
- **最大ドローダウン**（最重視）
- その他補助: 勝率、プロフィットファクター、平均保有日数

### 7.3 LLM判断のメタ検証
- ライブ運用で蓄積したLLM判断ログを対象に、「LLM確信度 vs 実績」の乖離を継続モニタ（確信度キャリブレーション）

---

## 8. レポート出力

### 8.1 配信方法
- **ローカル PDF / HTML 生成**（自分で開く）
- 併せて **CLI / ターミナル出力** でも確認可能
- メール・Slack等は対象外（MVPでは）

### 8.2 レポート内容（日次）

1. **市場レジーム判定結果**（Risk On/Off）
2. **新規エントリー推奨**
   - 銘柄・根拠・ストップ／ターゲット価格
   - 反証シナリオ（定量／定性トリガー一覧）
   - Bull/Bearディベートログの要旨
3. **保有銘柄のステータスとアクション**
   - ホールド／利食／損切 の判断
   - 反証トリガーのヒット状況
4. **クローズ銘柄のPostMortem**（該当日のみ）
   - 予想 vs 結果、学び
5. **（「本日は新規推奨なし」という結論も正常な出力として明示）**

## 12. ヒアリング回答サマリ（参照用）

- 利用者: 個人投資家（自分専用）
- AI役割: 売買シグナル生成（買い/売り判断）
- 発注: 提案のみ（手動）

- データ源優先: EDINET + TDnet 主軸、株価はまず無料→J-Quants
